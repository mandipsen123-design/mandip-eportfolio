HTML
Structure

HyperText Markup Language defines the skeleton of every web page — the raw, semantic content that browsers, screen readers, and search engines all rely on.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"/> <title>My Page</title> </head> <body> <header> <h1>Hello, World</h1> </header> <main> <p>Content goes here.</p> </main> <footer>© 2025</footer> </body> </html>
Tag Anatomy

Dissecting an HTML Element

Every HTML element follows the same pattern — an opening tag, optional content, and a closing tag. Some tags are self-closing.

<a href="https://example.com">Click here</a>

</a> Closing tag
Semantic HTML

Tags That Mean Something

Semantic tags tell browsers and developers what a piece of content actually is — not just what it looks like. This improves accessibility, SEO, and code readability.

<header>

Introductory content or navigation. Usually at the top of a page or section.

<nav>

A set of navigation links — site menus, breadcrumbs, pagination.

<main>

The dominant, unique content of the page — only one per document.

<article>

Self-contained, independently distributable content — a blog post, news story.

<section>

A thematic grouping of content, typically with a heading.

<aside>

Content tangentially related to the main content — sidebars, callouts.

<footer>

Closing content — copyright, links, author info for a page or section.

<figure>

Self-contained media (image, diagram, code) with an optional caption.

HTML Forms

User Input Elements

Forms are how HTML collects data from users. The <form> element groups inputs, and each input has a name, type, and placeholder to guide the user.


The form on the right uses: <input type="text">, <input type="email">, <select>, <textarea>, and <button type="submit">.
