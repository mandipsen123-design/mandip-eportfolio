The Stack

Three Layers, One Application

A "stack" is the combination of technologies used to build a complete web application. Each layer has a specific role, and full-stack developers understand all three.

Frontend

Everything a user sees and interacts with — rendered in the browser. The UI, layouts, animations, and client-side logic.

HTML CSS JavaScript React Vue
🖥️
⚙️

Backend

Server-side logic that processes requests, enforces business rules, and returns data to the frontend via APIs.

Node.js Python PHP REST API GraphQL

Database

Persistent storage for all application data — users, content, orders. The long-term memory of the application.

MySQL PostgreSQL MongoDB Redis
🗄️

Career Paths

Developer Roles

🎨

Frontend Developer

Specialises in UI/UX, visual design, browser APIs, and user interaction. Masters HTML, CSS, and JavaScript frameworks.

🔧

Backend Developer

Builds APIs, manages databases, handles authentication, and ensures the server-side logic is secure and scalable.

⚡

Full Stack Developer

Works across all layers. Can build an entire feature from database schema to polished UI — the most versatile role.

Live Example

A Request in Code

This demonstrates how a frontend fetch call maps to a backend handler, which queries a database and returns JSON data.

full-stack-example · frontend.js & server.js

🖥️ Frontend (browser)

// User clicks "Load Profile"
async function loadProfile(userId) {
  const res = await fetch(
    `/api/users/${userId}`
  );

  const data = await res.json();

  // Update the DOM
  document
    .querySelector('#name')
    .textContent = data.name;
}

loadProfile(42);

⚙️ Backend Response (JSON)

{
  "id":    42,
  "name":  "Priya Sharma",
  "role":  "Full Stack Dev",
  "posts": 128,
  "active": true,
  "stack": [
    "React",
    "Node.js",
    "PostgreSQL"
  ]
}

The backend queries the database with SELECT * FROM users WHERE id = 42, serialises the row to JSON, and returns it with status 200 OK.
